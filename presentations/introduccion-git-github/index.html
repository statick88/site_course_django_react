Introducción a Git y Github 🕹️

¿Qué vamos a aprender en este tutorial? 🤔

En este tutorial, aprenderás todo sobre Git y GitHub, desde conceptos básicos hasta cómo utilizarlo en un proyecto real.

¿Qué es Git? 🤔

Git es un sistema de control de versiones distribuido, creado por Linus Torvalds en 2005. Se utiliza para realizar un seguimiento de los cambios en el código fuente durante el desarrollo de software.

¿Por qué utilizar Git? 🤔

Git es ampliamente utilizado por desarrolladores de software de todo el mundo, desde estudiantes hasta grandes empresas tecnológicas. Es una herramienta fundamental para el desarrollo colaborativo y la gestión de proyectos de software.

Cómo se utiliza Git? 🤔

Git se utiliza mediante la línea de comandos o a través de interfaces gráficas de usuario. Proporciona comandos para realizar operaciones como inicializar un repositorio, realizar cambios, revisar historial, fusionar ramas, entre otros.

¿Para qué sirve Git? 🤔

Git sirve para realizar un seguimiento de los cambios en el código fuente, coordinar el trabajo entre varios desarrolladores, revertir cambios no deseados, y mantener un historial completo de todas las modificaciones realizadas en un proyecto.

Por qué utilizar Git? 🤔

Git ofrece varias ventajas, como la capacidad de trabajar de forma distribuida, la gestión eficiente de ramas para desarrollar nuevas funcionalidades o corregir errores sin afectar la rama principal, y la posibilidad de colaborar de forma efectiva con otros desarrolladores.

¿Dónde puedo utilizar Git? 🤔

Git puede ser utilizado en cualquier sistema operativo, incluyendo Windows, macOS y Linux. Además, es compatible con una amplia variedad de plataformas de alojamiento de repositorios, siendo GitHub una de las más populares.

  1. Visita el sitio web oficial de Git en https://git-scm.com/downloads.
  2. Descarga el instalador adecuado para tu sistema operativo y sigue las instrucciones de instalación.

Configuración 🛠️

Una vez instalado Git, es necesario configurar tu nombre de usuario y dirección de correo electrónico. Esto se puede hacer mediante los siguientes comandos: 

												
							git config --global user.name "Tu Nombre"
							git config --global user.email

  • Crea una nueva carpeta para tu proyecto y ábrela en Visual Studio Code.
  • Crea un archivo Python llamado hello_world.py y escribe el siguiente código:

Código

						
def welcome_message():
		name = input("Ingrese su nombre: ")
		print("Bienvenio,", name, "al curso de Django y React!")
														
if __name__ == "__main__":

Inicializar el repositorio

  • Inicializa un repositorio Git en la carpeta con el comando:
    •

    • Añade el archivo al área de preparación con:
      •

Inicializar el repositorio

Realiza un commit de los cambios con un mensaje descriptivo:

Comandos Básicos de Git 📝

git init: Inicializa un nuevo repositorio Git.

						
git init

Comandos Básicos de Git 📝

git add : Añade un archivo al área de preparación. 

							
	git add hello_world.py

Comandos Básicos de Git 📝

git commit -m "mensaje": Realiza un commit de los cambios con un mensaje descriptivo.

						
git commit -m "Añadir archivo hello_world.py"

Comandos Básicos de Git 📝

git push: Sube los cambios al repositorio remoto.

						
git push

Comandos Básicos de Git 📝

git pull: Descarga cambios del repositorio remoto.

						
git pull

Comandos Básicos de Git 📝

git branch: Lista las ramas disponibles.

						
git branch

Comandos Básicos de Git 📝

git checkout nombre_rama: Cambia a una rama específica.

						
git checkout nombre_rama

Comandos Básicos de Git 📝

git merge nombre_rama: Fusiona una rama con la rama actual. 

						
git merge nombre_rama

Comandos Básicos de Git 📝

git reset archivo: Descarta los cambios en un archivo. 

						
git reset archivo

Comandos Básicos de Git 📝

git diff: Muestra las diferencias entre ramas. 

						
git diff rama1 rama2

Estados en Git 📊

Local: Representa los cambios que realizas en tu repositorio local antes de hacer un commit. Estos cambios están únicamente en tu máquina.

Staging: Indica los cambios que has añadido al área de preparación con el comando git add. Estos cambios están listos

Estados en Git 📊

Commit: Son los cambios que has confirmado en tu repositorio local con el comando git commit. Estos cambios se han guardado de manera permanente en tu repositorio local.

Server: Son los cambios que has subido al repositorio remoto con el comando git push. Estos cambios están disponibles para otros colaboradores del proyecto.

Ejemplo Práctico: Moviendo Cambios entre Estados en Git 📝

Hemos creado un repositorio de Git para nuestro proyecto helloWorld en Python, pero cometimos un error al escribir el mensaje de bienvenida como "Bienvenio" en lugar de "Bienvenido". Además, queremos mejorar el mensaje de "Ingrese su nombre" a "Por favor ingrese su nombre: ".

Mover Cambios de Local a Staging:

Abre el archivo hello_world.py en tu editor de código. Encuentra la línea que dice print("Bienvenio Mundo") y cámbiala por print("Bienvenido Mundo").

Encuentra la línea que dice nombre = input("Ingrese su nombre: ") y cámbiala por nombre = input("Por favor ingrese su nombre: ").

Código

Actualmente tenemos el siguiente código: 

						
		
def welcome_message():
		name = input("Ingrese su nombre: ")
		print("Bienvenio,", name, "al curso de Django y React!")
		
if __name__ == "__main__":
welcome_message()

Código

Debemos cambiarlo por el siguiente código: 

									
def welcome_message():
		name = input("Por favor ingrese su nombre: ")
		print("Bienvenido,", name, "al curso de Django y React!")

if __name__ == "__main__":
		welcome_message()

Guarda los cambios.

Agregar Cambios de Local a Staging:

Para agregar los cambios realizados al área de staging, utiliza el comando: 

						
git add nombre_archivo

para añadir un archivo específico o

						
git add .

para añadir todos los archivos

Vovler a Local:

Si deseas deshacer los cambios de (staged) y volver al estado anterior (local), puedes usar el comando:

						
git checkout -- nombre_archivo

para deshacer los cambios en un archivo específico o

						
git checkout -- hello_world.py

para deshacer todos los cambios en todos los archivos

Mover los cambios de Statging a Commit

Una vez que estés satisfecho con los cambios y los hayas añadido al área de staging, puedes confirmarlos en un commit utilizando el comando:

						
git commit -m "Mensaje descriptivo

un ejemplo: 

										
git commit -m "Corregir mensaje de bienvenida y mejorar mensaje de ingreso"

para deshacer todos los cambios en todos los archivos

Estados en Git 📊

  • Local: Representa los cambios que realizas en tu repositorio local antes de hacer un commit. Estos cambios están únicamente en tu máquina.
  • Staging: Indica los cambios que has añadido al área de preparación con el comando git add. Estos cambios están listos para ser confirmados en el próximo commit.

Estados en Git 📊

  • Commit: Son los cambios que has confirmado en tu repositorio local con el comando git commit. Estos cambios se han guardado de manera permanente en tu repositorio local.
  • Server: Son los cambios que has subido al repositorio remoto con el comando git push. Estos cambios están disponibles para otros colaboradores del proyecto.
