Comandos básicos de Docker 1/2

En esta sección, aprenderemos los comandos básicos de Docker para interactuar con contenedores e imágenes.

  • docker run
  • docker ps
  • docker images

docker run 1/2

El comando docker run se utiliza para crear y ejecutar contenedores a partir de imágenes de Docker. Puedes especificar opciones y argumentos adicionales para personalizar el comportamiento del contenedor durante su ejecución. 

docker run nombre_de_la_imagen [opciones]

docker run 2/2

Por ejemplo, para crear y ejecutar un contenedor basado en la imagen de Ubuntu, puedes ejecutar: docker run ubuntu.

docker ps 1/2

El comando docker ps se utiliza para mostrar una lista de contenedores en ejecución en el sistema. Proporciona información básica sobre cada contenedor, como su ID, nombre, estado, puertos expuestos, etc. 

docker ps [opciones]

docker ps 2/2

Por ejemplo, para mostrar todos los contenedores en ejecución junto con información detallada, puedes ejecutar:

docker ps -a

docker images 1/2

El comando docker images se utiliza para mostrar una lista de imágenes disponibles en el sistema. Proporciona información sobre cada imagen, como su repositorio, etiqueta, ID y tamaño. 

docker images [opciones]

Por ejemplo, para mostrar todas las imágenes disponibles en el sistema, puedes ejecutar:

docker images

Comandos básicos de Docker 2/2

Otros comandos básicos de Docker incluyen:

  • docker pull
  • docker stop
  • docker rm
  • docker rmi

docker pull

El comando docker pull se utiliza para descargar una imagen de Docker desde un registro de imágenes, como Docker Hub, a tu sistema local. 

docker pull nombre_de_la_imagen

Por ejemplo, para descargar la imagen oficial de Ubuntu, puedes ejecutar:

docker pull ubuntu

docker stop

El comando docker stop se utiliza para detener uno o más contenedores en ejecución. 

docker stop ID_del_contenedor

Por ejemplo, para detener un contenedor con ID abcdef123456, puedes ejecutar:

docker stop abcdef123456

docker rm

El comando docker rm se utiliza para eliminar uno o más contenedores detenidos. 

docker rm ID_del_contenedor

Por ejemplo, para eliminar un contenedor con ID abcdef123456, puedes ejecutar:

docker rm abcdef123456.

docker rmi

El comando docker rmi se utiliza para eliminar una o más imágenes de Docker del sistema local. 

docker rmi nombre_de_la_imagen

Por ejemplo, para eliminar la imagen de Ubuntu, puedes ejecutar:

docker rmi ubuntu.

Creación y gestión de contenedores

En esta sección, aprenderemos a crear y gestionar contenedores Docker.

  • Creación de un contenedor
  • Gestión de ciclos de vida del contenedor (start, stop, restart)
  • Conexión a un contenedor en ejecución
  • Exposición de puertos
  • Asignación de volúmenes

Creación de un contenedor

Para crear un contenedor en Docker, puedes utilizar el comando docker run, seguido del nombre de la imagen que deseas utilizar. 

docker run nombre_de_la_imagen

Por ejemplo, para crear un contenedor basado en la imagen de Ubuntu, puedes ejecutar:

docker run ubuntu.

Gestión de ciclos de vida del contenedor (start, stop, restart) 1/2

Una vez que has creado un contenedor en Docker, puedes gestionar su ciclo de vida utilizando los comandos docker start, docker stop y docker restart. 

docker start ID_del_contenedor
docker stop ID_del_contenedor
docker restart ID_del_contenedor

Gestión de ciclos de vida del contenedor (start, stop, restart) 2/2

Por ejemplo, para iniciar un contenedor con ID abcdef123456, puedes ejecutar:

docker start abcdef123456.

Conexión a un contenedor en ejecución

Para conectarte a un contenedor en ejecución y ejecutar comandos dentro de él, puedes utilizar el comando docker exec. 

docker exec -it ID_del_contenedor /bin/bash

Por ejemplo, para conectarte a un contenedor con ID abcdef123456, puedes ejecutar:

docker exec -it abcdef123456 /bin/bash.

Exposición de puertos

Para exponer puertos desde un contenedor Docker al sistema host, puedes utilizar la opción -p o --publish al ejecutar el comando docker run. 

docker run -p puerto_host:puerto_contenedor nombre_de_la_imagen

Por ejemplo, para exponer el puerto 80 del contenedor al puerto 8080 del sistema host, puedes ejecutar:

docker run -p 8080:80 nginx.

Asignación de volúmenes 1/2

Para asignar volúmenes a un contenedor Docker y persistir datos entre ejecuciones, puedes utilizar la opción -v o --volume al ejecutar el comando docker run. 

docker run -v ruta_host:ruta_contenedor nombre_de_la_imagen

Asignación de volúmenes 2/2

Por ejemplo, para asignar el directorio /var/www del sistema host al directorio /usr/share/nginx/html del contenedor, puedes ejecutar:

docker run -v /var/www:/usr/share/nginx/html nginx

Uso de Dockerfile para la creación de imágenes personalizadas

En esta sección, aprenderemos a crear imágenes personalizadas utilizando Dockerfile.

  • Sintaxis básica de Dockerfile
  • Agregar dependencias
  • Configuración del entorno de trabajo
  • Ejecución de comandos durante la construcción de la imagen

Sintaxis básica de Dockerfile 1/2

Dockerfile es un archivo de texto que contiene instrucciones que Docker utilizará para construir una imagen. La sintaxis básica de un Dockerfile incluye una serie de comandos que describen las acciones necesarias para configurar el entorno y ejecutar la aplicación dentro del contenedor.

Sintaxis básica de Dockerfile 2/2

Por ejemplo, un Dockerfile simple podría comenzar con una instrucción FROM para especificar la imagen base, seguida de otras instrucciones como RUN, COPY, EXPOSE y CMD para configurar y ejecutar la aplicación.

Agregar dependencias 1/2

En un Dockerfile, puedes agregar dependencias necesarias para tu aplicación utilizando la instrucción RUN. Por ejemplo, si tu aplicación requiere ciertas bibliotecas o herramientas, puedes instalarlas dentro del contenedor utilizando comandos como apt-get, yum, pip, etc.

Agregar dependencias 2/2

Por ejemplo, para instalar las dependencias de Python en un contenedor basado en Ubuntu, puedes usar:

RUN apt-get update && apt-get install -y python3

Configuración del entorno de trabajo

Puedes configurar el entorno de trabajo dentro del contenedor utilizando la instrucción WORKDIR. Esta instrucción establece el directorio de trabajo para las instrucciones siguientes en el Dockerfile.

Por ejemplo, para establecer el directorio de trabajo en /app, puedes usar:

WORKDIR /app

Ejecución de comandos durante la construcción de la imagen 1/2

Puedes ejecutar comandos durante la construcción de la imagen utilizando la instrucción RUN. Estos comandos se ejecutarán cuando se construya la imagen, permitiéndote realizar configuraciones adicionales, instalar software, copiar archivos, etc.

Ejecución de comandos durante la construcción de la imagen 2/2

Por ejemplo, para copiar archivos desde el sistema host al contenedor durante la construcción de la imagen, puedes usar:

COPY archivos /directorio_destino

Redes en Docker

En esta sección, exploraremos cómo configurar redes en Docker para conectar contenedores entre sí.

  • Tipos de redes en Docker
  • Creación de redes personalizadas
  • Conexión de contenedores en una red
  • Comunicación entre contenedores en redes diferentes

Tipos de redes en Docker 1/2

Docker admite varios tipos de redes que permiten a los contenedores comunicarse entre sí y con el mundo exterior. Algunos de los tipos de redes comunes en Docker incluyen la red de puente (bridge network), la red de host (host network), la red de superposición (overlay network), y la red de contenedor (container network).

Tipos de redes en Docker 2/2

Cada tipo de red tiene sus propias características y se utiliza en diferentes situaciones, dependiendo de los requisitos de conectividad de los contenedores.

Creación de redes personalizadas 1/2

Además de los tipos de redes predeterminados proporcionados por Docker, también puedes crear tus propias redes personalizadas utilizando el comando docker network create. 

docker network create nombre_de_la_red

Creación de redes personalizadas 2/2

Esto te permite configurar y gestionar redes específicas para tus aplicaciones y contenedores, y proporciona mayor control sobre la conectividad de la red en tu entorno Docker.

Conexión de contenedores en una red 1/2

Una vez que has creado una red en Docker, puedes conectar contenedores a esa red utilizando la opción --network al ejecutar el comando docker run. 

docker run --network nombre_de_la_red nombre_de_la_imagen

Conexión de contenedores en una red 2/2

Esto permite que los contenedores se comuniquen entre sí a través de la red especificada, lo que facilita la implementación de arquitecturas de microservicios y la gestión de aplicaciones distribuidas.

Comunicación entre contenedores en redes diferentes 1/2

Docker también proporciona la capacidad de comunicarse entre contenedores que se encuentran en redes diferentes utilizando la red de superposición (overlay network) y otras técnicas de enrutamiento.

Comunicación entre contenedores en redes diferentes 2/2

Esto permite construir aplicaciones distribuidas y escalables que pueden funcionar en entornos de múltiples hosts sin comprometer la conectividad de red.

Volúmenes en Docker

En esta sección, aprenderemos a utilizar volúmenes en Docker para persistir datos entre ejecuciones de contenedores.

  • Creación de volúmenes
  • Asignación de volúmenes a contenedores
  • Copia de datos entre volúmenes y el sistema host
  • Uso de volúmenes anónimos y nombrados

Creación de volúmenes

Los volúmenes en Docker se utilizan para persistir datos entre ejecuciones de contenedores. Puedes crear volúmenes utilizando el comando docker volume create. 

docker volume create nombre_del_volumen

Esto creará un volumen en Docker que puede ser utilizado por uno o más contenedores para almacenar y compartir datos de forma persistente.

Asignación de volúmenes a contenedores

Una vez que has creado un volumen en Docker, puedes asignarlo a uno o más contenedores utilizando la opción -v o --volume al ejecutar el comando docker run. 

docker run -v nombre_del_volumen:ruta_en_el_contenedor nombre_de_la_imagen

Esto permite que los contenedores accedan al volumen y persistan datos en él durante su ejecución.

Copia de datos entre volúmenes y el sistema host

Puedes copiar datos entre volúmenes y el sistema host utilizando comandos de Docker, como docker cp. 

docker cp archivo_o_directorio ruta_del_contenedor

Esto te permite transferir archivos y directorios entre el sistema host y los volúmenes de Docker de manera rápida y sencilla.

Uso de volúmenes anónimos y nombrados 1/2

Docker admite tanto volúmenes anónimos como volúmenes nombrados. Los volúmenes anónimos son gestionados por Docker de forma automática y no tienen un nombre asignado. Por otro lado, los volúmenes nombrados son creados explícitamente por el usuario y tienen un nombre asociado.

Uso de volúmenes anónimos y nombrados 2/2

Ambos tipos de volúmenes pueden ser útiles en diferentes situaciones, dependiendo de los requisitos específicos de tu aplicación y de cómo prefieras gestionar los datos persistentes en Docker.

Ejemplos 🚀

Veremos algunos ejemplos prácticos para entender mejor los conceptos de Docker:

  • Crear un contenedor simple
  • Montar un volumen
  • Conectar contenedores en una red

Ejemplo 1: Crear un contenedor simple

En este ejemplo, crearemos un contenedor Docker simple basado en una imagen existente, como Ubuntu, y ejecutaremos un comando dentro de él. 

docker run ubuntu echo "Hola mundo desde Docker"

Este comando creará un contenedor basado en la imagen de Ubuntu y ejecutará el comando echo "Hola mundo desde Docker" dentro del contenedor. Luego, el contenedor se detendrá automáticamente.

Ejemplo 2: Montar un volumen 1/2

En este ejemplo, montaremos un volumen en un contenedor Docker para persistir datos entre ejecuciones del contenedor. 

docker run -v /ruta/local:/ruta/contenedor ubuntu

Ejemplo 2: Montar un volumen 2/2

Este comando crea un contenedor basado en la imagen de Ubuntu y monta el directorio /ruta/local del sistema host en el directorio /ruta/contenedor del contenedor.

De esta manera, cualquier dato almacenado en ese directorio dentro contenedor estará disponible en el sistema host y persistiráincluso después de detener el contenedor.

Ejemplo 3: Conectar contenedores en una red 1/2

En este ejemplo, crearemos una red en Docker y conectaremos dos contenedores para que puedan comunicarse entre sí. 

docker network create mynetwork
docker run --network mynetwork --name container1 ubuntu
docker run --network mynetwork --name container2 ubuntu

Ejemplo 3: Conectar contenedores en una red 2/2

Estos comandos crearán una red llamada mynetwork en Docker. Luego, dos contenedores basados en la imagen de Ubuntu se conectarán a esta red utilizando la opción --network.

Ahora, los contenedores container1 y container2 pueden comunicarse entre sí a través de la red mynetwork.
